# ralplan

Use after research or grill has reduced ambiguity enough to plan.

1. Summarize the target result, constraints, open risks, and acceptance criteria.
2. If the current provider exposes a consensus-planning runtime, hand off with the brief and request review.
3. If no runtime exists, produce a reviewed-plan checklist and mark the runtime as unsupported/fallback.
4. Stop at an implementation-ready plan; do not implement in this skill.
